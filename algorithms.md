# Category Two: Algorithms and Data Structures

## 📄 Artifact Description
The artifact showcased here is the **SNHU Travel System**, originally developed in Java during my CS 250 course. Its primary purpose is to load and display a curated list of top vacation destinations.

## ⚖️ Justification for Inclusion
I selected this artifact for the Algorithms and Data Structures category because it showcases my ability to evaluate and improve computational efficiency in data retrieval. 
* **The Problem:** The original project used static arrays and hardcoded values, meaning the data was fixed in size and could not be searched or sorted efficiently.
* **The Solution:** I refactored the data layer to use a dynamic `ArrayList` for flexible data handling and implemented a custom algorithmic sorting mechanism.
* **New Skills Showcased:** This demonstrates my ability to select appropriate data structures based on memory and manipulation needs, as well as writing custom sorting logic rather than relying on basic sequential processing.

## 🎯 Alignment with Course Outcomes
This enhancement fulfills the planned objectives established early in the capstone:
* **Outcome 2 (Algorithmic Principles):** I applied core algorithmic principles to solve a specific data problem, ensuring that the travel system can handle larger datasets while maintaining operational speed.
* **Outcome 4 (Innovative Tools):** By transitioning away from basic static storage to dynamic collections, I used modern computing practices that deliver practical value and accomplish industry-specific goals.

## 💭 Reflection on the Process
Working on this artifact helped me bridge the gap between theoretical data structures and their practical application.
* **What I Learned:** I learned that small design choices at the data layer can have massive impacts on application performance as scaling occurs.
* **Challenges:** A challenge I faced was ensuring that the UI elements updated smoothly when the underlying sorted list changed. This forced me to evaluate the tight coupling between the display components and the data storage, ultimately making me a stronger developer.
